    Questions:

    1. What command can you type into your server’s terminal to check if apache is currently running?

      systemctl status apache2

    2. Apache implements the HTTP protocol. The protocol is extremely simple, with just a few commands. What is the difference between the GET, POST, and HEAD commands?

      • The GET command retrieves data from the server from a specified resource.
      • The POST command sends data to the server to update or create a specified resource.
      • The HEAD command is similar to the GET command, except that it does not return the data to the user. These are used to check what a GET command will return before making the GET request.

    3. What is the difference between ServerRoot and DocumentRoot?

      ServerRoot = path to the webserver executable/dir

      DocumentRoot = the directory path of your files that are delivered by the server

    4. What port does apache listen to for HTTP connections from browsers by default?

      The apache listens to port 80 for HTTP connections from browsers.

    5. What file on your server contains apache’s traffic logs? What about apache’s error logs? These files tend to get very long, with the most recent logged events being appended to the end of the file. On the terminal, how can you view the last few lines of any file?

      • /var/log/apache2/access.log
      • /var/log/apache2/error.log
      • The 'tail -n x filename' command outputs the last x lines of a file

    6. What is a Directory Index file? Why would it be desirable to have one in each directory?

      DirectoryIndex is used to allow you to land default page when client access a directory. For example, when a visitor request a directory to access it on your website, you can define the file to load when they request to a directory.

    7. How do you get apache to redirect users to a specified page upon receiving a 404?

      ErrorDocument 404 /<URL> will give a local URL to redirect to
      ErrorDocument 404 <URL> will give an external URL to redirect to (if the action is a valid URL).
      ErrorDocument 404 “Text” (if none of the above).

    8. How do you give every user on your Linux server their own web space?

      By running the following command: sudo a2enmod userdir. We created the module userdir.conf in the folder /etc/apache2/mods-available/ to enable users to have their own web space. Then under the dbteam user, we created a public_html folder to place html files
